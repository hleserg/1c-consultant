## ⚠️ Протокол работы (обязателен для каждой задачи)

**Метки**

- `[ТЫ]` в заголовке — задача владельца (Сергей): ручная (UI GitHub/Sentry/Devin, согласования, сбор данных). Агент её НЕ берёт.
- Без метки — для агентов-кодеров.

**Git-workflow (жёсткое правило, детали в `AGENTS.md`)**

1. На `main` → создать ветку ПЕРЕД работой. Никогда не коммитить в main напрямую.
2. В одной ветке может быть несколько задач — допустимо.
3. Закончил → PR (можно draft) → снять draft → запустить все проверки.
4. Дождаться ОКОНЧАНИЯ всех проверок и ВСЕХ ревьюверов (Devin, Sentry-ревью, любые «жёлтые»).
5. Исправить ВСЁ: комментарии ревью по пушу, ошибки CI, конфликты мерджа. Resolve починенных замечаний.
6. Запушить → повторять 4–5, пока: все проверки пройдены, ноль незакрытых замечаний, ноль ошибок CI, ноль конфликтов.
7. ТОЛЬКО при полном зелёном — мерджить. Не выходит → написать «не могу» с причиной и остановиться.

**Статусы и сдача**

- Взял задачу → `In Progress`. Не трогать задачу, чей блокер не в `Done`.
- Перед завершением: перечитать задание + DoD → перечитать что сделал → сверить каждый пункт DoD.
- Всё на 100% и уверен → комментарий-отчёт (что сделано, файлы, как проверено) → `In Review` (финальный `Done` ставит владелец).
- Частично / не уверен → `In Review` + в комментарии что НЕ сделано и почему, вопросы/риски. Не выдавать частичное за полное.

## Что строит проект (кратко)

Локальный MCP-агент — компетентный консультант по 1С ERP. Собирает контекст проблемы, проводит
настоящий ресёрч по источникам (офлайн-ИТС, выгрузка конфигурации, доки платформы), отвечает только
из найденного, сам определяет нехватку данных и идёт искать. Не «улучшенный поиск», а спец.

**Стек:** Python ≥3.11, uv, FastMCP, Claude (разговор + зрение для скринов + LLM-судья),
BGE-M3 через FlagEmbedding (гибрид dense+sparse, локально), Cohere Rerank v4 (фолбэк — локальный
BGE-reranker), OCR для картинок. Часть решений (финальная стратегия поиска, OCR vs мультимодал)
принимается ПО ДАННЫМ обкатки, не на старте.

**Источники (легальные):** выгрузка конфы в XML, офлайн-версия ИС 1С:ИТС (по подписке),
портал техдокументации платформы. Спецификации форматов — опора на cc-1c-skills (MIT).
Без скрейпинга закрытого ИТС.

**Реалистичная планка:** «полностью без фантазий» — направление, не выключатель. 100% гарантии нет.
Цель — устойчивое поведение «почти всегда сам идёт в источники и отвечает по делу».
